## 1.Linux 754 介绍

- Linux 文件权限分为读 `4`、写 `2`、执行 `1`。
- `754` 表示所有者 `7=rwx`，同组用户 `5=r-x`，其他用户 `4=r--`。

## 2.介绍冒泡排序，选择排序，冒泡排序如何优化

```js
function bubbleSort(params) {
  for (let i = 0; i < params.length - 1; i++) {
    for (let j = params.length - i; j > i + 1; j--) {
      let preVal = params[j]
      let nextVal = params[j - 1]
      if (preVal < nextVal) {
        params[j] = nextVal
        params[j - 1] = preVal
      }
    }
  }
  return params
}
function selectSort(params) {
  let minIndex
  for (let i = 0; i < params.length - 1; i++) {
    minIndex = i
    for (let j = i + 1; j < params.length; j++) {
      if (params[minIndex] > params[j]) {
        minIndex = j
      }
    }
    let minVal = params[minIndex]
    params[minIndex] = params[i]
    params[i] = minVal
  }
  return params
}
```

## 3.transform 动画和直接使用 left、top 改变位置有什么优缺点

- `left/top` 会改变布局位置，通常触发布局和重绘，性能开销较大。
- `transform` 多数情况下只触发合成，能利用 GPU，动画更流畅。
- `left/top` 适合真实布局变化，`transform` 更适合纯视觉位移和动画。

## 4.如何判断链表是否有环

- 使用快慢指针：慢指针每次走一步，快指针每次走两步。
- 如果存在环，两个指针最终会相遇；如果快指针走到 `null`，说明无环。

## 5.介绍二叉搜索树的特点

- 任意节点左子树所有值小于该节点，右子树所有值大于该节点。
- 中序遍历可以得到有序序列。
- 平衡时查找、插入、删除复杂度接近 `O(log n)`，退化时可能变成 `O(n)`。

## 6.介绍暂时性死区

- `let`、`const` 声明会被提升，但在声明语句执行前不能访问。
- 从作用域开始到声明完成之间的区域称为暂时性死区，访问会抛出 `ReferenceError`。

## 7.ES6 中的 map 和原生的对象有什么区别

- `Map` 的 key 可以是任意类型，对象的 key 主要是字符串或 Symbol。
- `Map` 保留插入顺序，并且有 `size`、`set`、`get`、`has` 等专用 API。
- 频繁增删键值对时，`Map` 通常更合适；普通结构化数据用对象更直观。

## 8.观察者和发布-订阅的区别

- 观察者模式中，被观察者直接维护观察者列表并通知它们。
- 发布-订阅模式中，发布者和订阅者通过事件中心解耦，彼此不直接知道对方。

## 9.react 异步渲染的概念,介绍 Time Slicing 和 Suspense

- 异步渲染指 React 可以把渲染工作拆分、暂停、恢复和丢弃，避免长任务阻塞主线程。
- Time Slicing 是把渲染任务切成小片段，在空闲时间继续执行。
- Suspense 用于等待异步资源时声明式展示 fallback，比如懒加载组件或数据加载。

## 10.16.X 声明周期的改变

- React 16.3 引入 `getDerivedStateFromProps` 和 `getSnapshotBeforeUpdate`。
- `componentWillMount`、`componentWillReceiveProps`、`componentWillUpdate` 被标记为不安全生命周期。
- 主要原因是异步渲染下这些生命周期可能被多次调用，容易产生副作用问题。

## 11.16.X 中 props 改变后在哪个生命周期中处理

- 需要根据 props 派生 state 时使用 `static getDerivedStateFromProps`。
- 需要执行副作用，比如请求、订阅、DOM 操作，应放在 `componentDidUpdate` 中处理。

## 12.介绍纯函数

- 相同输入永远得到相同输出。
- 不修改外部状态，也不产生副作用。
- Redux 的 `reducer` 就应该是纯函数。

## 13.前端性能优化

- 加载层面：代码分割、懒加载、压缩资源、CDN、缓存策略。
- 渲染层面：减少重排重绘、使用虚拟列表、合理使用 `transform` 动画。
- 代码层面：减少重复计算、防抖节流、避免不必要的组件渲染。

## 14.pureComponent 和 FunctionComponent 区别

- `PureComponent` 是类组件，内置对 `props` 和 `state` 的浅比较。
- `FunctionComponent` 是函数组件，本身没有浅比较能力，可配合 `React.memo`、`useMemo`、`useCallback` 优化。

## 15.介绍 JSX

- JSX 是 JavaScript 的语法扩展，用类似 HTML 的方式描述 UI。
- 编译后会变成 `React.createElement` 或新的 JSX runtime 调用。
- JSX 本质仍是 JS，可以在其中写表达式、传 props、组合组件。

## 16.如何做 RN 在安卓和 IOS 端的适配

- 使用平台判断，如 `Platform.OS`、`Platform.select`。
- 对尺寸、状态栏、安全区域、字体、权限、原生能力做差异处理。
- 复杂差异可以拆分为 `.ios.js` 和 `.android.js` 文件。

## 17.RN 为什么能在原生中绘制成原生组件（bundle.js）

- JS 代码会被打包成 `bundle.js`，在 JS 引擎中执行。
- React Native 将组件树转换为原生视图描述，再通过桥或 JSI/Fabric 与原生层通信。
- 最终渲染的是平台原生组件，而不是 WebView DOM。

## 18.介绍虚拟 DOM

- 虚拟 DOM 是用 JS 对象描述真实 DOM 或原生视图结构。
- 状态变化后生成新的虚拟 DOM，与旧树 diff 后得到最小更新。
- 优点是声明式、跨平台、便于批量更新和统一调度。

## 19.如何设计一个 localStorage，保证数据的实效性

- 存储数据时同时保存过期时间 `expire` 或时间戳。
- 读取时判断当前时间是否超过过期时间，过期则删除并返回空。
- 可封装 `set/get/remove`，并处理 JSON 序列化和异常兜底。

## 20.如何设计 Promise.all()

- 接收 promise 数组，返回一个新的 Promise。
- 按索引收集每个 promise 的成功结果，全部成功后 resolve 结果数组。
- 任意一个 promise reject 时，立即 reject 对应原因。

## 21.介绍高阶组件

- 高阶组件是接收组件并返回新组件的函数。
- 常用于逻辑复用，如权限控制、埋点、数据注入、错误边界包装。
- 注意透传 props、静态属性和 ref，否则容易影响原组件能力。

## 23.react 性能优化

- 使用 `React.memo`、`PureComponent`、`useMemo`、`useCallback` 减少无效渲染。
- 列表渲染使用稳定 `key`，大列表使用虚拟滚动。
- 合理拆分组件和状态，避免父组件更新导致大范围子组件重渲染。
