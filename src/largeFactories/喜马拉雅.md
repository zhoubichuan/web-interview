## 2.介绍 Promise

- Promise 用于表示异步操作的最终完成或失败。
- 它支持链式调用，能统一处理成功和失败回调。
- 构造函数同步执行，`then/catch` 回调进入微任务。

## 5.React 的生命周期

- 挂载卸载过程：constructor、componentWillMount、componentDidMount、componentWillUnmount
- 更新过程：componentWillReceiveProps、shouldComponentUpdate、componentWillUpdate、componentDidUpdate、render
- 新增生命周期：getDerivedStateFromProps、getSnapshotBeforeUpdate

## 6.componentWillReceiveProps 的触发条件是什么

- 父组件重新渲染并向子组件传入 props 时会触发，即使 props 值可能没变。
- 该生命周期已不推荐使用，可用 `getDerivedStateFromProps` 或 `componentDidUpdate` 替代。

## 7.React16.3 对生命周期的改变

- 新增 `getDerivedStateFromProps` 和 `getSnapshotBeforeUpdate`。
- `componentWillMount`、`componentWillReceiveProps`、`componentWillUpdate` 被标记为不安全。
- 目的是适配异步渲染，避免副作用生命周期被重复调用带来的问题。

## 8.介绍下 React 的 Filber 架构

- 正确写法是 Fiber。Fiber 是 React 的调和架构，把渲染拆成可中断的工作单元。
- 它支持优先级调度、暂停恢复和并发渲染。

## 9.画 Filber 渲染树

- Fiber 树通过 `child`、`sibling`、`return` 指针表示父子和兄弟关系。
- 每个 Fiber 节点对应一个组件、DOM 节点或文本节点。
- 当前屏幕对应 current 树，更新时构建 workInProgress 树。

## 10.介绍 React 高阶组件

- 高阶组件是接收组件并返回新组件的函数。
- 常用于权限、埋点、数据注入、逻辑复用。
- 注意透传 props、ref 和静态属性。

## 11.父子组件之间如何通信

- 父传子通过 props。
- 子传父通过父组件传入回调函数。
- 跨层级可用 Context 或状态管理工具。

## 12.Redux 怎么实现属性传递，介绍下原理

- Redux store 保存全局状态。
- React-Redux 订阅 store，通过 selector 或 `mapStateToProps` 把状态映射为组件 props。
- dispatch 后 reducer 生成新状态，订阅组件收到通知并更新。

## 13.React-Router 版本号

- 早期常见 v3/v4 差异较大，v4 开始以组件化路由为核心。
- 当前项目应以实际依赖版本为准，关注 `BrowserRouter`、`Routes/Route`、hooks 等 API 差异。

## 14.网站 SEO 怎么处理

- 使用服务端渲染 SSR 或静态生成 SSG。
- 设置合理 title、description、语义化 HTML、sitemap、结构化数据。
- 对 SPA 可做预渲染或关键页面服务端直出。

## 15.介绍下 HTTP 状态码

- 2xx 请求成功
  - 200 请求成功并且返回正确的网页
  - 201 新建或修改成功
  - 202 删除成功
- 3xx 每次重定向的次数不超过 5 次
  - 304 走网页缓存，节省宽带和开销
- 4xx 请求出错
  - 400 请求出错，服务器不理解请求的语法
  - 401 没有权限
  - 403 有权限，但访问被禁止
  - 404 没有找到页面
- 5xx 请求出错
  - 500 请求出错
  - 503 服务器暂时无法访问（宕机，停机维护）

## 16.403、301、302 是什么

- 403：服务器理解请求，但拒绝访问。
- 301：永久重定向，搜索引擎会更新索引。
- 302：临时重定向，资源位置暂时变化。

## 1.介绍 HTTPS

- HTTPS 是在 HTTP 和 TCP 之间加入 TLS/SSL 加密层。
- 它提供加密传输、身份认证和完整性校验。

## 1.HTTPS 怎么建立安全通道

- 客户端和服务端协商 TLS 版本、加密套件和随机数。
- 服务端发送证书，客户端校验证书合法性。
- 双方通过非对称加密或密钥交换算法协商会话密钥，之后用对称加密传输数据。

## 1.前端性能优化（JS 原生和 React）

- JS 原生层面减少长任务、事件代理、防抖节流、缓存计算结果。
- React 层面减少无效渲染、组件懒加载、虚拟列表、合理使用 memo。
- 资源层面压缩、缓存、CDN、按需加载。

## 1.用户体验做过什么优化

- 首屏 loading、骨架屏、错误兜底、空状态。
- 表单即时校验、防重复提交、操作反馈。
- 离线缓存、弱网提示、滚动位置恢复。

## 1.对 PWA 有什么了解

是一个能安装到本地并且可以离线缓存的网页，通过一些新的技术手段使得安装到本地的应用能够获取类似原生应用的体验，并且拥有安装体积小（加载一个网页大小），可以在线使用不强制安装，跨平台（iOS、Android、PC）等优点

## 1.对安全有什么了解

- 常见风险包括 XSS、CSRF、点击劫持、中间人攻击、敏感信息泄露。
- 防护包括输入输出转义、CSP、SameSite Cookie、HTTPS、权限校验和安全响应头。

## 1.介绍下数字签名的原理

- 发送方对内容计算摘要，再用私钥加密摘要生成签名。
- 接收方用公钥解密签名并对比摘要。
- 可以证明内容未被篡改，并确认发送方身份。
