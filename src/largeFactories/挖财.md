# 十三、挖财

## 1.工作收获

- 可以从业务理解、工程化能力、协作能力和问题排查能力几个角度回答。
- 面试中建议结合具体项目，说明遇到的问题、方案和结果。

## 2.JS 的原型

- 每个函数都有 `prototype`，实例通过原型链访问构造函数原型上的属性和方法。
- 原型链用于属性查找和方法复用，也是 JS 继承的基础。

## 3.变量作用域链

- 函数执行时会创建执行上下文，并保存外层词法环境引用。
- 查找变量时先查当前作用域，找不到再沿作用域链向外查找。

## 4.call、apply、bind 的区别

- 三者都可以改变函数执行时的 `this`。
- `call` 参数逐个传，`apply` 参数用数组传。
- `bind` 返回一个新函数，不会立即执行。

## 5.防抖和节流的区别

```
function throttle(fn,wait){
  var timer
  return function(){
    var context = this
    var args = arguments
    if(!timer){
      timer = setTimeout(()=>{
        timer =null
        fn.apply(this,args)
      },wait)
    }
  }
}

  var timer
  return function(){
    var context =this
    var args = arguments

    timer = setTimeout(()=>{
```

## 6.介绍各种异步方案

- 回调函数、Promise、Generator、async/await。
- 浏览器层面还有事件监听、定时器、网络请求等异步 API。
- async/await 可读性最好，Promise 更适合并发组合。

## 7.react 生命周期

- 挂载：`constructor`、`render`、`componentDidMount`。
- 更新：`getDerivedStateFromProps`、`shouldComponentUpdate`、`render`、`componentDidUpdate`。
- 卸载：`componentWillUnmount`。

## 8.介绍 Fiber

- Fiber 是 React 的新调和架构，把渲染任务拆成可中断的小单元。
- 每个 Fiber 节点对应组件树中的一个工作单元。
- 它支持优先级调度、暂停恢复和并发渲染。

## 9.前端性能优化

- 资源优化：压缩、缓存、CDN、按需加载。
- 渲染优化：减少重排重绘、虚拟列表、合理使用合成层。
- JS 优化：防抖节流、减少长任务、避免内存泄漏。

## 10.介绍 DOM 树对比

- 前端框架通常先生成新虚拟 DOM，再与旧虚拟 DOM diff。
- 同层比较节点类型和 key，判断复用、移动、删除或创建。
- 最后把差异提交到真实 DOM。

## 11.react 中的 key 的作用

- `key` 用于标识列表项身份，帮助 React 在 diff 中复用节点。
- 稳定 key 可以减少不必要更新，并避免组件状态错位。

## 12.如何设计状态树

- 按业务领域拆分模块，避免所有状态堆在一个对象里。
- 状态尽量扁平化，减少深层嵌套更新成本。
- 区分服务端数据、UI 状态和派生数据。

## 13.介绍 css，xsrf

- CSS 负责页面样式和布局，核心包括盒模型、选择器、布局、动画等。
- XSRF/CSRF 是利用用户登录态伪造请求的攻击。
- 防护方式包括 CSRF Token、SameSite Cookie、Origin/Referer 校验。

## 14.http 缓存控制

1.强制缓存 ：命中就会从本地获取缓存资源，没有命中就会判断是否命中协商缓存 expires、cache-control 2.协商缓存：如果命中，服务器将请求返回但是不会返回资源，而是告诉客户端直接从缓存中获取 last-modified、if-modified-since、etag、if-none-match

## 15.项目中如何应用数据结构

- 树：菜单、路由、组织架构、DOM。
- 队列：任务调度、请求并发控制。
- Map/Set：缓存、去重、索引。
- 栈：撤销重做、表达式解析、路由历史。

## 16.native 提供了什么能力给 RN

- 原生 UI 组件、系统 API、传感器、相机、定位、支付、推送等能力。
- RN 通过 Native Module 或原生组件桥接这些能力。

## 17.如何做工程上的优化

- 规范目录结构、代码规范、自动化测试和 CI。
- 构建优化、按需加载、缓存策略和发布回滚。
- 监控日志、错误上报、性能指标和质量门禁。

## 18.shouldComponentUpdate 是为了解决什么问题

提高性能

## 19.如何解决 props 层级过深的问题

- 使用 Context 或状态管理库减少逐层透传。
- 通过组件组合把需要的数据放到更接近使用位置的地方。
- 也可以拆分容器组件和展示组件。

## 20.前端怎么做单元测试

- 使用 Jest/Vitest 测试纯函数和工具方法。
- React 组件可用 Testing Library 测试渲染和交互。
- 网络请求、时间、浏览器 API 需要 mock。

## 21.webpack 生命周期

- 主要包括初始化参数、创建 compiler、开始编译、构建模块、生成 chunk、输出资源。
- plugin 通过 Tapable hooks 介入不同阶段。

## 22.webpack 打包的整个过程

- 从入口开始递归解析依赖，构建模块依赖图。
- loader 转换模块内容，plugin 扩展编译流程。
- 根据依赖关系生成 chunk，最后输出 bundle 和静态资源。

## 23.常用的 plugins

- `HtmlWebpackPlugin` 生成 HTML。
- `MiniCssExtractPlugin` 抽离 CSS。
- `DefinePlugin` 注入环境变量。
- `CopyWebpackPlugin` 复制静态资源。

## 24.pm2 怎么做进程管理，进程挂掉怎么处理

- pm2 可以守护 Node 进程，异常退出后自动重启。
- 支持 cluster 模式、多实例、日志管理和监控。
- 可配置最大重启次数、内存阈值和健康检查。

## 25.不用 pm2 怎么做进程管理

- 使用 Node `cluster` 模块自己管理 worker。
- 使用 systemd、Docker、Kubernetes 等外部进程管理工具。
- 需要实现异常监听、自动拉起、日志和优雅退出。
