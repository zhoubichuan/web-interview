# 三、饿了么

## 1.小程序里面开页面最多多少

- 微信小程序页面栈最多通常是 10 层。
- 超过后应使用 `redirectTo`、`reLaunch` 或 `switchTab` 控制页面栈。

## 2.React 子父组件之间如何传值

- 父传子通过 `props`。
- 子传父通过父组件传给子组件的回调函数。
- 跨层级可用 Context 或状态管理库。

## 3.Emit 事件怎么发，需要引入什么

- Vue 中可用 `$emit('eventName', data)` 触发父组件监听的事件。
- Node 或普通 JS 可使用 EventEmitter/EventBus，需要引入或实现事件发布订阅模块。

## 4.介绍下 React 高阶组件，和普通组件有什么区别

- 高阶组件就是一个函数，且该函数接受一个组件作为参数，并返回一个新的组件，它只是一种模式，这种模式是由 react 自身的组合性质必然产生的。
- 我们将它们称为纯组件，因为它们可以接受任何动态提供的子组件，但它们不会修改或复制其输入组件中的任何行为。

## 5.一个对象数组，每个子对象包含一个 id 和 name，React 如何渲染出全部的 name

- 使用 `array.map` 渲染列表，并把稳定的 `id` 作为 `key`。
- 示例：`list.map(item => <div key={item.id}>{item.name}</div>)`。

## 6.在哪个生命周期里写

- 类组件中请求数据通常写在 `componentDidMount`。
- 函数组件中使用 `useEffect(() => {}, [])`。

## 7.其中有几个 name 不存在，通过异步接口获取，如何做

- 先渲染已有数据，对缺失 name 的项发起异步请求。
- 请求完成后合并回列表状态并重新渲染。
- 多个缺失项可用 `Promise.all` 并发获取。

## 8.渲染的时候 key 给什么值，可以使用 index 吗，用 id 好还是 index 好

- 优先使用稳定唯一的 `id`。
- 不建议用 index，因为插入、删除、排序会导致组件复用错乱。
- 只有列表完全静态且不会重排时才可考虑 index。

## 9.webpack 如何配 sass，需要配哪些 loader

- 常见配置需要 `sass-loader`、`sass`、`css-loader`、`style-loader`。
- 生产环境通常用 `MiniCssExtractPlugin.loader` 替代 `style-loader` 抽离 CSS。

## 10.配 css 需要哪些 loader

- css-loader
- style-loader

## 11.如何配置把 js、css、html 单独打包成一个文件

- JS 通过 webpack 入口和 output 输出。
- CSS 使用 `MiniCssExtractPlugin` 抽离为独立 CSS 文件。
- HTML 使用 `HtmlWebpackPlugin` 生成并自动注入资源。

## 12.div 垂直水平居中（flex、绝对定位）

- flex：父元素设置 `display:flex; justify-content:center; align-items:center;`。
- 绝对定位：子元素 `position:absolute; top:50%; left:50%; transform:translate(-50%, -50%);`。

## 13.两个元素块，一左一右，中间相距 10 像素

- 可用 flex：父元素 `display:flex; gap:10px;`。
- 也可以左元素 `margin-right:10px`，或用 grid 设置列间距。

## 14.上下固定，中间滚动布局如何实现

- 父容器使用 flex 纵向布局，高度 `100vh`。
- 头尾固定高度，中间区域 `flex:1; overflow:auto;`。

## 15.[1, 2, 3, 4, 5]变成[1, 2, 3, a, b, 5]

```js
let arr = [1, 2, 3, 4, 5].splice(3, 1, "a", "b")
```

## 16.取数组的最大值（ES5、ES6）

```js
Math.max.apply(null, arr)
Math.max(...arr)
```

## 17.apply 和 call 的区别

apply 第二个参数为数组
call 第二个参数及以后为 arg1,arg2...等形式

## 18.ES5 和 ES6 有什么区别

es6 中添加了很多新功能，其实大部分是 es5 的语法糖

## 19.some、every、find、filter、map、forEach 有什么区别

- some 只要有一项符合条件就返回 true
- every 每一项符合条件才返回 true
- find 查找符合要求的某一项并返回
- filter 筛选出符合条件的数组项，返回一个新数组，对原数组不会造成影响
- map 修改符合要求的数组项并返回整个数组
- forEach 循环遍历整个数组

## 20.上述数组随机取数，每次返回的值都不一样

- 可以先打乱数组，再依次取值。
- 或维护一个已取索引集合，每次随机后判断是否重复。
- 数据量较小时推荐洗牌算法。

## 21.如何找 0-5 的随机数，95-99 呢

```js
Math.random() * 5
Math.random() * 5 + 95
```

## 22.页面上有 1 万个 button 如何绑定事件

使用事件委托的方式绑定

## 23.如何判断是 button

- 事件委托中可判断 `event.target.tagName === 'BUTTON'`。
- 更稳妥可使用 `event.target.closest('button')`，兼容点击到按钮内部元素的情况。

## 24.页面上生成一万个 button，并且绑定事件，如何做（JS 原生操作 DOM）

使用事件委托和节流处理

## 25.循环绑定时的 index 是多少，为什么，怎么解决

- 使用 `var` 循环绑定时，回调执行时拿到的往往是循环结束后的最终 index。
- 原因是 `var` 是函数作用域，共享同一个变量。
- 可用 `let`、闭包或给元素写入 `data-index` 解决。

## 26.页面上有一个 input，还有一个 p 标签，改变 input 后 p 标签就跟着变化，如何处理

- 监听 input 输入事件，读取 `input.value` 后更新 `p.textContent`。
- 框架中可用受控组件或双向绑定实现。

## 27.监听 input 的哪个事件，在什么时候触发

- 通常监听 `input` 事件，输入内容实时变化时触发。
- `change` 事件通常在失焦或提交变更后触发，实时性较弱。
