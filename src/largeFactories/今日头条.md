# 四、今日头条

## 1.对 async、await 的理解，内部原理

- `async/await` 是 Promise 的语法糖，让异步流程以同步形式书写。
- `async` 函数返回 Promise，`await` 会暂停当前函数，等待 Promise 完成后把后续代码放入微任务继续执行。
- 内部思想类似 Generator + 自动执行器。

## 3.清除浮动

- 父元素使用 clearfix：通过 `::after` 创建块级元素并 `clear: both`。
- 也可以让父元素触发 BFC，如 `overflow: hidden`。
- 现代布局优先使用 flex/grid，减少浮动布局。

## 4.定位问题（绝对定位、相对定位等）

- `relative` 相对自身原位置偏移，不脱离文档流。
- `absolute` 相对最近定位祖先偏移，脱离文档流。
- `fixed` 相对视口固定，`sticky` 根据滚动阈值切换相对和固定。

## 5.从输入 URL 到页面加载全过程

- URL 解析、DNS 查询、建立 TCP 连接，HTTPS 还要 TLS 握手。
- 发送 HTTP 请求，服务器处理并返回响应。
- 浏览器解析 HTML，构建 DOM/CSSOM，生成渲染树，布局、绘制、合成。

## 6.tcp3 次握手

- 客户端发送 SYN。
- 服务端返回 SYN + ACK。
- 客户端再发送 ACK，连接建立。
- 目的是确认双方收发能力正常。

## 7.tcp 属于哪一层（1 物理层 -> 2 数据链路层 -> 3 网络层(ip)-> 4 传输层(tcp) -> 5 应用层(http)）

- TCP 属于传输层。
- 它提供可靠、有序、面向连接的字节流传输。

## 8.redux 的设计思想

- 单一数据源，整个应用状态放在 store 中。
- state 只读，只能通过 dispatch action 描述变更。
- 使用纯函数 reducer 根据旧 state 和 action 生成新 state。

## 9.接入 redux 的过程

- 创建 reducer 和 store。
- 使用 Provider 把 store 注入 React 应用。
- 组件通过 `connect` 或 hooks 获取状态并 dispatch action。

## 10.绑定 connect 的过程

- 定义 `mapStateToProps` 把 state 映射为 props。
- 定义 `mapDispatchToProps` 或直接使用 dispatch。
- 调用 `connect(mapStateToProps, mapDispatchToProps)(Component)` 返回增强组件。

## 11.connect 原理

- `connect` 通过 Context 获取 store。
- 订阅 store 更新，执行 selector 得到组件需要的 props。
- 当映射结果变化时触发包装组件重新渲染。

## 12.webpack 介绍

- webpack 是模块打包工具，从入口递归构建依赖图。
- loader 负责转换模块，plugin 扩展构建流程。
- 最终输出 bundle、chunk 和静态资源。

## 13.== 和 ===的区别，什么情况下用相等==

== 在比较时只比较大小，===会比较数据类型

## 14.bind、call、apply 的区别

bind 函数不会立即执行，第一个参数是修改的 this 对象，后面的都是参数
call 和 apply 函数会立即执行，第一参数是修改的 this 对象，第二个参数 call 后面是 arg1,arg2,arg3...的形式，apply 是数组形式

## 15.动画的了解

- CSS 动画包括 `transition` 和 `animation`。
- JS 动画可用 `requestAnimationFrame` 按帧更新。
- 性能上优先使用 `transform`、`opacity`，减少触发布局和重绘的属性。

## 16.介绍下原型链（解决的是继承问题吗）

- 原型链是对象属性查找机制：自身找不到就沿原型继续查找。
- 它可以实现继承和方法复用，但不仅仅服务于继承。

## 17.对跨域的了解

- 浏览器的同源策略造成跨越
- 跨域请求数据其实已经到达浏览器了，之所以前端页面拿不到数据是因为浏览器把信息屏蔽掉了，可以通过降级浏览器的方式解除这种限制拿到非同源请求的数据，常用于前端开发
- 后端请求是不存在跨域的
- 跨域的几种方式
  - jsonp：通过 script 或者 img 等标签不受浏览器同源策略影响的标签实现跨域，具体是前端先写好函数将参数传给相关页面，将函数和相关前端需要请求的参数通过 js 传递给后端，后端通过 js 解析参数，拿到前端函数名称，后端执行这个函数传入相关参数，前端将后端传过来的 js 插入到页面中，执行了后端的函数，前端通过之前的函数拿到后端的参数
  - cors：后端设置请求头允许跨域
  - proxymiddle:代理常见 webpack 中的代理，实际上是用 node 框架中 express 的代理服务
  - postmessage:h5 的通信方式支持跨域
  - websockit:消息推送，支持跨域
  - domian:二级域名之间的通信
  - hash：通过二级域名之间通过 hash 传值
  - name：二级域名之间通过 name 跨域
  - nginx：代理跨域
