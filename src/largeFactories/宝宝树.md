# 宝宝树

## 1.使用 canvas 绘图时如何组织成通用组件

- 把画布初始化、尺寸适配、绘制流程、事件处理封装到组件内部。
- 外部通过 props 传入数据和配置，通过回调暴露交互事件。
- 注意高清屏缩放、重绘时机和资源销毁。

## 2.formData 和原生的 ajax 有什么区别

表单请求会刷新整个页面，原生 ajax 只会局部刷新页面

## 3.介绍下表单提交，和 formData 有什么关系

- 表单提交可以由浏览器原生提交，也可以用 Ajax 提交。
- `FormData` 用于构造表单键值对，支持文件上传。
- Ajax 提交 FormData 时无需手动拼接 multipart 数据。

## 4.介绍 redux 接入流程

- 创建 reducer 和 store。
- 使用 Provider 注入 store。
- 组件通过 `connect` 或 hooks 读取 state、dispatch action。

## 5.rudux 和全局管理有什么区别（数据可控、数据响应）

- Redux 有统一数据流和 reducer 约束，状态变化可追踪。
- 全局变量可任意修改，缺少订阅通知和调试能力。
- Redux 更新后能通知组件响应式渲染。

## 6.RN 和原生通信

- JS 调原生通过 Native Module 或原生组件桥接。
- 原生调 JS 可通过事件、回调或消息机制。
- 新架构中也可通过 JSI/Fabric 提升通信效率。

## 7.介绍 MVP 怎么组织

- Model 负责数据和业务逻辑。
- View 负责展示。
- Presenter 负责协调 View 和 Model，处理交互逻辑。
- 目的是降低视图和业务逻辑耦合。

## 10.koa2 中间件原理

- Koa2 中间件基于 async/await 和洋葱模型。
- 请求先按注册顺序进入中间件，遇到 `await next()` 后进入下一个。
- 下游执行完成后再回到上游继续执行。

## 11.常用的中间件

- `koa-router` 路由。
- `koa-bodyparser` 解析请求体。
- `koa-static` 静态资源。
- 日志、鉴权、错误处理、跨域中间件。

## 12.服务端怎么做统一的状态处理

- 使用统一响应结构，如 `{ code, data, message }`。
- 用错误处理中间件捕获异常并映射错误码。
- 对鉴权、参数校验、日志记录做统一封装。

## 13.如何对相对路径引用进行优化

- 配置 webpack/Vite alias，如 `@` 指向 `src`。
- TypeScript 项目同步配置 `paths`。
- 避免过深的 `../../../`，提升可读性和可维护性。

## 14.node 文件查找优先级

- 相对路径先按文件查找，再尝试补扩展名。
- 如果是目录，会查找 `package.json` 的 `main` 字段，再查 `index.js`。
- 非相对模块会从当前目录开始逐级向上查找 `node_modules`。
