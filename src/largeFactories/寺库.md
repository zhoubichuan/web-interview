# 十二、寺库

## 1.介绍 Promise 的特性，优缺点

- Promise 有三种状态，状态一旦改变不可逆。
- 优点是链式调用、统一错误处理，能缓解回调地狱。
- 缺点是默认无法取消，复杂流程仍可能写出较长链路。

## 2.介绍 Redux

- Redux 是集中式状态管理库，通过 store 保存状态。
- 组件 dispatch action，reducer 根据 action 返回新 state。
- 状态更新可预测、可追踪，适合复杂共享状态。

## 3.RN 的原理，为什么可以同时在安卓和 IOS 端运行

- RN 用 JS 描述组件树，再映射为平台原生组件。
- 业务逻辑和组件描述跨平台复用，平台差异由原生渲染层和桥接层处理。

## 4.RN 如何调用原生的一些功能

- 通过 Native Module 暴露原生方法给 JS 调用。
- UI 组件可通过原生 ViewManager 封装后导出给 RN 使用。

## 5.介绍 RN 的缺点

- 复杂动画和高性能场景可能需要原生优化。
- 原生模块维护成本较高，升级版本可能有兼容问题。
- 包体、启动性能和平台差异需要额外治理。

## 6.介绍排序算法和快排原理

- 常见排序有冒泡、选择、插入、归并、快速排序等。
- 快排选择基准值，把数组分为小于和大于基准的两部分，再递归排序。
- 平均复杂度 `O(n log n)`，最坏情况 `O(n^2)`。

## 7.堆和栈的区别

栈存放基本数据类型
堆存放引用数据类型

## 8.介绍闭包

- 闭包是函数可以访问其定义时外层作用域变量的机制。
- 内部函数被外部引用时，外层变量仍然可达，不会被回收。

## 9.闭包的核心是什么

变量常驻内存，形成不销毁的作用域

## 10.网络的五层模型

- 应用层、传输层、网络层、数据链路层、物理层。
- HTTP 属于应用层，TCP/UDP 属于传输层，IP 属于网络层。

## 11.HTTP 和 HTTPS 的区别

- HTTPS = HTTP + TLS/SSL。
- HTTPS 提供加密传输、身份认证和完整性校验。
- HTTP 明文传输，容易被窃听和篡改。

## 12.HTTPS 的加密过程

- 客户端校验证书，确认服务端身份。
- 双方协商会话密钥。
- 后续数据使用对称加密传输，并通过摘要保证完整性。

## 13.介绍 SSL 和 TLS

- SSL/TLS 是用于网络安全传输的协议层。
- TLS 是 SSL 的后续标准，现代 HTTPS 实际使用 TLS。
- 它提供加密、认证和完整性保护。

## 14.介绍 DNS 解析

- DNS 把域名解析为 IP 地址。
- 浏览器会依次查浏览器缓存、系统缓存、hosts、递归 DNS。
- 最终通过根域名、顶级域名、权威 DNS 找到目标 IP。

## 15.JS 的继承方法

- 原型链继承、构造函数继承、组合继承、寄生组合继承。
- ES6 使用 `class extends`，本质仍是基于原型链。

## 16.介绍垃圾回收

- JS 垃圾回收主要基于可达性分析。
- 从全局对象和调用栈出发可访问到的对象保留，不可达对象被回收。
- 常见算法是标记清除，现代引擎还会做分代和增量回收。

## 17.cookie 的引用为了解决什么问题

- HTTP 是无状态协议，Cookie 用来在客户端保存状态标识。
- 服务端可通过 Cookie 识别用户、维持登录态或保存偏好。

## 18.cookie 和 localStorage 的区别

cookie 服务端产生
localStorage 存放本地

## 19.如何解决跨域问题

- 降级浏览器
- jsonp
- cors
- nginx
- node 中间件 -proxy-middleWare
- postMessage
- websockit
- document.domain
- document.name
- location.hash

## 20.前端性能优化

- 减少资源体积和请求数：压缩、合并、懒加载、CDN。
- 利用缓存：强缓存、协商缓存、Service Worker。
- 优化渲染：减少重排重绘、虚拟列表、避免长任务。
