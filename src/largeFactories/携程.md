# 十六、携程

## 1.手写两道算法题

- 面试常见可准备数组去重、数组扁平化、二分查找、快排、防抖节流。
- 回答时说明思路、复杂度和边界条件。

## 2.对 React 看法，有没有遇到一些坑

- React 用组件化和状态驱动视图提升了复杂 UI 的可维护性。
- 常见坑有 state 异步更新、key 使用 index、闭包拿到旧 state、过度渲染。
- 可通过函数式更新、稳定 key、memo 和合理状态拆分解决。

## 3.对闭包的看法，为什么要用闭包

- 闭包让函数保留定义时的外层作用域变量。
- 可用于封装私有变量、缓存状态、防抖节流和柯里化。
- 需要注意长期引用大对象可能造成内存占用。

## 4.手写数组去重函数

```js
function unique(arr) {
  return Array.from(new Set(arr))
}
```

## 5.手写数组扁平化函数

```js
function flatten(arr) {
  return arr.reduce((res, item) => {
    return res.concat(Array.isArray(item) ? flatten(item) : item)
  }, [])
}
```

## 6.介绍下 Promise 的用途和性质

- Promise 用于管理异步结果，解决回调嵌套和错误传递问题。
- 它有 pending、fulfilled、rejected 三种状态，状态一旦改变不可逆。
- 支持链式调用和组合方法，如 `all`、`race`、`allSettled`。

## 7.Promise 和 Callback 有什么区别

- Callback 简单直接，但多层嵌套会导致回调地狱。
- Promise 支持链式调用、状态管理和统一错误捕获。
- Promise 回调属于微任务，普通回调执行时机取决于具体 API。

## 8.React 生命周期

- 挂载：`constructor`、`render`、`componentDidMount`。
- 更新：`getDerivedStateFromProps`、`shouldComponentUpdate`、`render`、`componentDidUpdate`。
- 卸载：`componentWillUnmount`。
