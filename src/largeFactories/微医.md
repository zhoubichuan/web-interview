# 十五、微医

## 1.介绍 JS 数据类型，基本数据类型和引用数据类型的区别

- 基本类型有 `undefined`、`null`、`boolean`、`number`、`string`、`symbol`、`bigint`。
- 引用类型主要是对象，包括普通对象、数组、函数、日期、正则等。
- 基本类型保存值本身，赋值会复制值；引用类型保存引用地址，赋值会复制引用。

## 2.Array 是 Object 类型吗

- 是。数组是特殊对象，`typeof []` 返回 `object`。
- 判断数组应使用 `Array.isArray()` 或 `Object.prototype.toString.call(arr)`。

## 3.数据类型分别存在哪里

var a = {name: "前端开发"}; var b = a; a = null 那么 b 输出什么
var a = {b: 1} 存放在哪里
var a = {b: {c: 1}}存放在哪里

- `b` 仍然指向原对象 `{name: "前端开发"}`，`a = null` 只是断开了 `a` 的引用。
- 对象本体存放在堆内存中，变量保存的是对象引用。
- 嵌套对象也是对象，内层对象同样在堆中，通过引用关联。

## 4.栈和堆的区别

- 栈内存空间较小、访问快，主要存放执行上下文、基本类型值和对象引用。
- 堆内存空间较大，主要存放对象等引用类型数据，由垃圾回收器管理。

## 5.垃圾回收时栈和堆的区别

- 栈中变量随函数执行上下文销毁自动释放。
- 堆中对象需要通过可达性判断，无法从根对象访问到的对象会被垃圾回收。

## 6.数组里面有 10 万个数据，取第一个元素和第 10 万个元素的时间相差多少

- 正常数组通过下标访问，时间复杂度都是 `O(1)`，理论上差别很小。
- 实际性能可能受数组稀疏程度、引擎优化和内存布局影响。

## 7.栈和堆具体怎么存储

- 基本类型变量直接在栈中保存值。
- 引用类型变量在栈中保存引用地址，对象内容保存在堆中。
- 多个变量可以保存同一个堆对象的引用。

## 8.介绍闭包以及闭包为什么没清除

- 闭包是函数可以访问其定义时词法作用域中的变量。
- 如果内部函数仍被外部引用，相关外层变量仍可达，所以不会被垃圾回收。

## 9.闭包的使用场景

- 封装私有变量、缓存中间状态、实现柯里化。
- 防抖节流中保存定时器。
- 异步回调中保留当前作用域数据。

## 10.JS 怎么实现异步

- 浏览器或 Node 提供异步 API，如定时器、网络请求、事件监听、文件 IO。
- JS 主线程通过事件循环调度宏任务和微任务，实现非阻塞执行。

## 11.异步整个执行周期

- 同步代码先进入调用栈执行。
- 异步任务由宿主环境处理，完成后把回调放入任务队列。
- 调用栈清空后，事件循环先清空微任务，再执行下一个宏任务。

## 12.Promise 的三种状态

- `pending`：等待中。
- `fulfilled`：已成功。
- `rejected`：已失败。
- 状态只能从 `pending` 转为成功或失败，之后不可再改变。

## 13.Async/Await 怎么实现

- `async/await` 是 Generator 和 Promise 的语法糖。
- `async` 函数返回 Promise，`await` 会等待 Promise 状态变化后继续执行后续代码。

## 14.Promise 和 setTimeout 执行先后的区别

- Promise 的 `then` 回调属于微任务。
- `setTimeout` 回调属于宏任务。
- 同一轮事件循环中，同步代码执行完后会先执行微任务，再执行下一个宏任务。

## 15.JS 为什么要区分微任务和宏任务

- 微任务用于在当前宏任务结束后尽快处理状态收尾，比如 Promise 回调。
- 宏任务用于调度下一轮事件循环，比如定时器、IO、UI 事件。
- 区分两者可以让异步任务有更细的优先级和更稳定的执行顺序。

## 16.Promise 构造函数是同步还是异步执行，then 呢

- Promise 构造函数中的代码是同步执行的。
- `then`、`catch`、`finally` 的回调会进入微任务队列异步执行。

## 17.发布-订阅和观察者模式的区别

- 观察者模式中，被观察者直接通知观察者。
- 发布-订阅模式中，发布者和订阅者通过事件中心通信，解耦程度更高。

## 18.JS 执行过程中分为哪些阶段

- 编译阶段：创建词法环境、变量提升、函数声明提升。
- 执行阶段：逐行执行代码，处理作用域链、调用栈和 this。
- 异步阶段：通过事件循环调度宏任务和微任务。

## 19.词法作用域和 this 的区别

- 词法作用域由代码书写位置决定，函数定义时基本确定。
- `this` 多数情况下由调用方式决定，运行时才确定。
- 箭头函数没有自己的 `this`，会继承外层 `this`。

## 20.平常是怎么做继承

- ES5 常用寄生组合继承：构造函数继承属性，原型链继承方法。
- ES6 使用 `class extends`，本质仍然基于原型链。

## 21.深拷贝和浅拷贝

- 浅拷贝只复制一层，嵌套对象仍共享引用。
- 深拷贝会递归复制所有层级，拷贝后对象互不影响。
- 常见方式有递归、`structuredClone`、`lodash.cloneDeep`。

## 22.loadsh 深拷贝实现原理

- `lodash.cloneDeep` 会递归遍历对象属性并创建新对象。
- 它会处理数组、对象、Map、Set、Date、RegExp 等多种类型。
- 对循环引用会用缓存记录已拷贝对象，避免无限递归。

## 23.ES6 中 let 块作用域是怎么实现的

- JS 引擎会为块级作用域创建新的词法环境。
- `let/const` 绑定在该词法环境中，离开块后不可访问。
- 声明前处于暂时性死区，访问会报错。

## 24.React 中 setState 后发生了什么

- React 会把更新放入更新队列，计算新的 state。
- 之后触发调和过程，生成新的虚拟 DOM 并与旧树 diff。
- 最后把变化提交到真实 DOM，并执行相关生命周期或 effect。

## 25.setState 为什么默认是异步

- React 会批量合并多次状态更新，减少重复渲染。
- 异步调度也方便 React 根据优先级安排更新，提高性能。

## 26.setState 什么时候是同步的

- 在 React 18 之前，原生事件、定时器、Promise 回调中的 `setState` 往往表现为同步。
- React 18 开启自动批处理后，大多数场景都会批量更新；需要同步刷新可用 `flushSync`。

## 27.为什么 3 大框架出现以后就出现很多 native（RN）框架（虚拟 DOM）

- 虚拟 DOM 把 UI 描述抽象成 JS 对象，不强依赖浏览器 DOM。
- 这让同一套声明式组件模型可以映射到不同平台的原生组件。
- RN、Weex 等框架就是把虚拟节点转换为原生视图。

## 28.虚拟 DOM 主要做了什么

- 用 JS 对象描述 UI 结构。
- 状态变化时生成新树，与旧树 diff。
- 计算出最小变更并提交给真实渲染层。

## 29.虚拟 DOM 本身是什么（JS 对象）

- 虚拟 DOM 本质是普通 JS 对象，描述节点类型、属性、子节点等信息。
- React 中对应的结构通常称为 React Element。

## 30.304 是什么

- 304 表示协商缓存命中，资源未修改。
- 浏览器可直接使用本地缓存，响应体为空。
- 常由 `If-None-Match/ETag` 或 `If-Modified-Since/Last-Modified` 触发。

## 31.打包时 Hash 码是怎么生成的

- webpack 会根据文件内容、模块依赖或 chunk 内容计算 hash。
- 内容变化会导致对应 hash 改变，方便浏览器缓存失效。
- 常见有 `hash`、`chunkhash`、`contenthash`。

## 32.随机值存在一样的情况，如何避免

- 随机值不能完全保证唯一，应结合时间戳、递增序号、业务前缀或 UUID。
- 分布式场景可使用雪花算法、数据库唯一约束等方式兜底。

## 33.使用 webpack 构建时有无做一些自定义操作

- 可以自定义 loader 处理特殊文件转换。
- 可以写 plugin 监听 webpack hooks，实现资源分析、构建提示、上传 CDN 等能力。
- 也可以自定义 splitChunks、alias、externals、环境变量注入等配置。

## 34.webpack 做了什么

- 从入口文件开始分析依赖图。
- 通过 loader 转换不同类型模块。
- 使用 plugin 扩展构建流程。
- 最终输出浏览器可运行的 bundle、chunk 和静态资源。

## 35.a，b 两个按钮，点击 aba，返回顺序可能是 baa，如何保证是 aba（Promise.then）

- 给每次点击生成递增序号，响应回来后按序号排队处理。
- 或把请求串行化，前一个请求完成后再执行下一个。
- 如果只关心最后一次结果，则丢弃旧请求结果。

## 36.node 接口转发有无做什么优化

- 使用连接池或 keep-alive 复用连接。
- 增加超时、重试、熔断、降级和缓存。
- 对日志、错误码、鉴权和参数校验做统一处理。

## 37.node 起服务如何保证稳定性，平缓降级，重启等

- 使用 pm2、cluster 或容器多进程部署。
- 进程异常自动拉起，重启前停止接收新请求并等待存量请求完成。
- 配合健康检查、限流、熔断、日志监控和告警。

## 38.RN 有没有做热加载

- 开发环境支持 Fast Refresh，可以在保留状态的情况下刷新 JS 代码。
- 线上热更新通常通过 CodePush 等方案下发 JS bundle，但原生代码变更仍需发版。

## 39.RN 遇到的兼容性问题

- 常见问题包括 Android/iOS 样式差异、权限差异、状态栏和安全区域差异。
- 还包括不同系统版本、机型尺寸、原生模块 API 差异。
- 通常通过 `Platform`、条件样式和原生适配层解决。

## 40.RN 如何实现一个原生的组件

- 原生侧实现 ViewManager 或 Native Module。
- JS 侧通过桥接导出组件或方法。
- 然后在 RN 中像普通组件一样引入并传递 props。

## 41.RN 混原生和原生混 RN 有什么不同

- RN 混原生：主应用以 RN 为主，部分能力调用原生模块。
- 原生混 RN：主应用是原生工程，部分页面嵌入 RN 页面。
- 两者主要差异在路由管理、包体组织、通信方式和发布流程。

## 42.什么是单页项目

- 单页项目 SPA 只有一个 HTML 入口，页面切换由前端路由控制。
- 优点是切换流畅、状态保留好；缺点是首屏和 SEO 需要额外优化。

## 43.遇到的复杂业务场景

- 可以从权限控制、复杂表单、流程编排、实时数据、跨端适配等角度回答。
- 面试时重点说明业务难点、技术方案、取舍、结果和复盘。

## 44.Promise.all 实现原理

- 接收可迭代对象，返回一个新的 Promise。
- 并行执行所有 Promise，按原顺序收集结果。
- 全部成功则 resolve 结果数组，任意一个失败则立即 reject。
