# 五、滴滴

## 1.react-router 怎么实现路由切换

- `hash` 模式通过修改 `location.hash` 并监听 `hashchange`。
- `history` 模式通过 `pushState/replaceState` 修改 URL，并监听 `popstate`。
- URL 变化后匹配路由配置，渲染对应组件。

## 2.react-router 里的 Link 标签和 a 标签有什么区别

- `a` 标签会触发浏览器默认跳转，可能导致页面整体刷新。
- `Link` 会阻止默认行为，调用路由 API 更新地址并触发组件切换。

## 3.a 标签默认事件禁掉之后做了什么才实现了跳转

- 调用 `history.pushState` 或修改 `hash` 更新 URL。
- 通知路由系统重新匹配路径并更新 React 组件。

## 4.React 层面的性能优化

- 使用 `React.memo`、`PureComponent`、`useMemo`、`useCallback`。
- 合理拆分组件和状态，避免父组件更新导致无关子组件重渲染。
- 长列表使用虚拟滚动，列表项使用稳定 `key`。

## 5.整个前端性能提升大致分几类

- 加载性能：压缩、缓存、CDN、懒加载、代码分割。
- 渲染性能：减少重排重绘、虚拟列表、动画使用 `transform`。
- 运行性能：防抖节流、减少重复计算、优化内存和事件绑定。

## 6.import { Button } from 'antd' ，打包的时候只打包 button，分模块加载，是怎么做到的

- 通过 ES Module 静态分析和 tree shaking 删除未使用模块。
- 组件库也可配合 `babel-plugin-import` 按需转换为具体组件路径。
- 样式也需要按需引入，避免全量 CSS 被打包。

## 7.使用 import 时，webpack 对 node_modules 里的依赖会做什么

- webpack 会从入口递归分析 `import` 依赖，构建模块依赖图。
- 对 `node_modules` 中被引用的模块同样会解析、转换并打入 bundle。
- 可通过 `externals`、splitChunks、tree shaking 等方式优化依赖体积。

## 8.JS 异步解决方案的发展历程以及优缺点

- 回调函数简单直接，但容易形成回调地狱。
- Promise 支持链式调用和统一错误处理，但复杂流程仍会有较多 `then`。
- Generator 可暂停恢复，但需要执行器；`async/await` 可读性最好，本质是 Promise 语法糖。

## 9.Http 报文的请求会有几个部分

- 请求行：方法、URL、协议版本。
- 请求头：描述客户端、缓存、内容类型等元信息。
- 空行：分隔 header 和 body。
- 请求体：POST/PUT 等请求携带的数据。

## 10.cookie 放哪里，cookie 能做的事情和存在的价值

在服务端生成，存放在客户端，存放一些标识判断客户端身份，如免密登录，设置过期时间，当长时间不登录设置的标识会消失

## 11.cookie 和 token 都存放在 header 里面，为什么只劫持前者

- Cookie 会被浏览器自动携带，容易被 CSRF 利用。
- Token 通常由前端手动放入 `Authorization`，第三方站点无法自动带上。
- 如果 Token 存在 localStorage，也要防 XSS 窃取。

## 12.cookie 和 session 有哪些方面的区别

- Cookie 存在客户端，Session 数据主要存在服务端。
- Cookie 每次请求自动携带，大小有限；Session 通过 session id 关联用户数据。
- Cookie 可设置过期时间、域、路径、SameSite、HttpOnly 等属性。

## 13.React 中 Dom 结构发生变化后内部经历了哪些变化

- 状态变化触发 render，生成新的虚拟 DOM。
- React 对新旧虚拟 DOM 做 diff，找出需要更新的部分。
- commit 阶段把变更应用到真实 DOM，并执行生命周期或 effect。

## 14.React 挂载的时候有 3 个组件，textComponent、composeComponent、domComponent，区别和关系，Dom 结构发生变化时怎么区分 data 的变化，怎么更新，更新怎么调度，如果更新的时候还有其他任务存在怎么处理

- 文本节点、原生 DOM 节点和自定义组件会被抽象成不同类型的 Fiber 节点。
- 数据变化会产生更新对象并进入更新队列，React 根据优先级调度渲染。
- diff 时根据节点类型和 `key` 判断复用、移动、删除或新建。
- 有更高优先级任务时，低优先级渲染可以被打断，之后再继续或重做。

## 15.key 主要是解决哪一类的问题，为什么不建议用索引 index（重绘）

- `key` 用于帮助 React 在列表 diff 中识别节点身份。
- 使用 index 时，插入、删除、排序会导致节点身份错乱，可能引发错误复用和额外更新。
- 应优先使用稳定唯一 id。

## 16.Redux 中异步的请求怎么处理

- Redux 本身只处理同步 action。
- 异步请求通常交给中间件，如 `redux-thunk`、`redux-saga`、`redux-observable`。
- 请求完成后再 dispatch 成功或失败 action 更新状态。

## 17.Redux 中间件是什么东西，接受几个参数（两端的柯里化函数）

- Redux 中间件是增强 `dispatch` 的函数。
- 形式通常是 `store => next => action => {}`。
- 可以在 action 到 reducer 之前做日志、异步请求、埋点、错误处理等。

## 18.柯里化函数两端的参数具体是什么东西

- 第一层接收 `store`，可拿到 `getState` 和 `dispatch`。
- 第二层接收 `next`，表示下一个中间件或原始 dispatch。
- 第三层接收 `action`，即当前要处理的动作。

## 19.中间件是怎么拿到 store 和 action，然后怎么处理

- `applyMiddleware` 创建 store 时会把 `storeAPI` 传给每个中间件。
- dispatch action 时，中间件链按顺序执行。
- 中间件可以拦截、修改、延迟 action，也可以调用 `next(action)` 继续传递。

## 20.state 是怎么注入到组件的，从 reducer 到组件经历了什么样的过程

- reducer 返回新 state 后，store 通知订阅者。
- React-Redux 订阅 store，并通过 selector 或 `mapStateToProps` 取出组件需要的数据。
- 数据通过 props 或 hook 返回值注入组件，触发组件重新渲染。

## 21.koa 中 response.send、response.rounded、response.json 发生了什么事，浏览器为什么能识别到它是一个 json 结构或是 html

- Koa 原生主要通过 `ctx.body` 设置响应体，Express 才有常见的 `send/json`。
- 框架会根据响应内容设置 `Content-Type`，如 `application/json` 或 `text/html`。
- 浏览器根据 `Content-Type` 和响应内容决定如何解析。

## 22.koa-bodyparser 怎么来解析 request

- 监听请求流的 `data` 和 `end`，收集请求体内容。
- 根据 `Content-Type` 解析 JSON、表单等格式。
- 解析结果挂到 `ctx.request.body`，供后续中间件使用。

## 23.webpack 整个生命周期，loader 和 plugin 有什么区别

- webpack 从初始化、编译、构建模块、生成 chunk、输出文件等阶段组成。
- loader 负责把某类文件转换成 JS 或 webpack 可处理的模块。
- plugin 基于 hooks 扩展整个构建流程，能力更广。

## 24.介绍 AST（Abstract Syntax Tree）抽象语法树

- AST 是源代码解析后的树形结构，每个节点代表一种语法结构。
- Babel、TypeScript、ESLint、webpack loader 都会使用 AST 做分析和转换。

## 25.安卓 Activity 之间数据是怎么传递的

- 常见方式是通过 `Intent` 携带 extras 传递基本数据或序列化对象。
- 复杂数据可使用全局状态、数据库、文件、ViewModel 或事件总线。

## 26.安卓 4.0 到 6.0 过程中 WebView 对 js 兼容性的变化

- 不同 Android 版本 WebView 内核和 JS 能力不同，低版本对新语法和新 API 支持较差。
- Android 4.4 后 WebView 基于 Chromium，兼容性和调试能力明显提升。
- 实际项目通常需要 Babel/polyfill 和能力检测兜底。

## 27.WebView 和原生是如何通信

- JS 调原生：通过 `JavascriptInterface`、URL scheme、postMessage 等方式。
- 原生调 JS：通过 `evaluateJavascript` 或加载 `javascript:` URL。
- 通信时要做好参数序列化、回调管理和安全白名单。

## 28.跨域怎么解决，有没有使用过 Apache 等方案

降级浏览器、cors、jsonp、websocket、nginx、proxy-middleware、postmessage、hash、name、domain
