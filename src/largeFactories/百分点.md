# 七、百分点

## 2.重新渲染 render 会做些什么
- 会对新旧vnode进行比对
## 3.哪些方法会触发 react 重新渲染

- setState,但是setState传入null时，并不会触发render
- 父组件重新渲染，即使是传入的子组件props未发生变化
## 4.state 和 props 触发更新的生命周期分别有什么区别

- state 更新通常由组件自身调用 `setState` 触发。
- props 更新来自父组件重新渲染并传入新 props。
- 两者都会进入更新流程，区别主要在数据来源不同；副作用通常放在 `componentDidUpdate`。

## 5.setState 是同步还是异步

- setState(obj)异步的
- setState(()=>{})异步的

## 6.对无状态组件的理解
它是为了创建纯展示组件，这种组件只负责根据传入的props来展示，不涉及到state状态的操作，组件不会被实例化，整体渲染新年得到提升，不能访问this对象，不能访问生命周期的方法
## 7.介绍 Redux 工作流程

- 组件 dispatch action。
- reducer 接收旧 state 和 action，返回新 state。
- store 保存新状态并通知订阅者。
- 组件重新获取状态并渲染。

## 8.介绍 ES6 的功能

- let const 定义的变量不会提升
- promise、generator 、async await 解决回调地狱
- 箭头函数

## 9.let、const 以及 var 的区别

- `var` 是函数作用域，存在变量提升，可重复声明。
- `let` 和 `const` 是块级作用域，有暂时性死区，不可重复声明。
- `const` 声明后不能重新赋值，但对象内部属性仍可修改。

## 10.浅拷贝和深拷贝的区别

- 浅拷贝只拷贝第一层的数据，里面的数据都是引用数据不是真正的拷贝
- 深拷贝是把嵌套的对象也拷贝一份

## 11.介绍箭头函数的 this

- 箭头函数没有 this，所以他的 this 执行外部的 this

## 12.介绍 Promise 和 then

- Promise 表示异步操作结果，有 pending、fulfilled、rejected 三种状态。
- `then` 用于注册成功和失败回调，并返回新的 Promise，支持链式调用。
- `then` 回调会作为微任务执行。

## 14.算法：前 K 个最大的元素

- 小数据可以先排序，再取前 K 个，复杂度 `O(n log n)`。
- 更优方案是维护大小为 K 的小顶堆，遍历数组时比堆顶大就替换。
- 小顶堆方案复杂度约为 `O(n log k)`。
