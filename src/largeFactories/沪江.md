# 十、沪江

## 1.介绍下浏览器跨域

- 跨域是浏览器同源策略限制导致的，同源要求协议、域名、端口都相同。
- 限制主要针对 JS 读取跨域响应，不是禁止请求发出。

## 2.怎么去解决跨域问题

- 常用方案有 CORS、JSONP、反向代理、postMessage、WebSocket。
- 开发环境可用 webpack devServer proxy，生产环境常用网关或 Nginx 代理。

## 3.jsonp 方案需要服务端怎么配合

- 前端通过 `script` 标签请求，并传递 callback 名称。
- 服务端返回 `callback(data)` 形式的 JS 代码。
- JSONP 只支持 GET，且需要注意 XSS 风险。

## 4.Ajax 发生跨域要设置什么（前端）

- 前端如果需要携带 Cookie，要设置 `withCredentials = true` 或 fetch 的 `credentials`。
- 服务端必须返回匹配的 CORS 头，前端单独设置无法绕过同源策略。

## 5.加上 CORS 之后从发起到请求正式成功的过程

- 简单请求会直接发送，浏览器检查响应中的 CORS 头。
- 非简单请求会先发送 OPTIONS 预检。
- 预检通过后再发送真实请求，响应头符合规则时浏览器才把结果交给 JS。

## 6.xsrf 跨域攻击的安全性问题怎么防范

- 使用 SameSite Cookie、CSRF Token、Referer/Origin 校验。
- 重要操作避免只依赖 Cookie 身份，增加二次校验。
- 对写操作使用非简单请求也能提高攻击成本。

## 7.使用 Async 会注意哪些东西

- `await` 会暂停当前 async 函数，多个无依赖任务不要串行 await。
- 错误要用 `try/catch` 或在外层捕获。
- 注意 async 函数返回值始终是 Promise。

## 8.Async 里面有多个 await 请求，可以怎么优化（请求是否有依赖）

- 无依赖请求用 `Promise.all` 并发执行。
- 有依赖请求再按顺序 `await`。
- 可结合并发池限制同时请求数量。

## 9.Promise 和 Async 处理失败的时候有什么区别

- Promise 链用 `.catch()` 捕获失败。
- async/await 通常用 `try/catch` 捕获，更接近同步写法。
- 未捕获的 async 错误会变成返回 Promise 的 rejected 状态。

## 10.Redux 在状态管理方面解决了 React 本身不能解决的问题

- React 自身状态适合局部组件，跨层级、跨页面共享会变复杂。
- Redux 提供统一 store、可预测更新流程和调试能力。
- 适合复杂全局状态、多人协作和状态可追踪场景。

## 11.Redux 有没有做过封装

- 常见封装包括统一 action 类型、请求状态、错误处理和 loading 管理。
- 可以封装 selector、模块化 reducer、异步中间件和业务 hooks。

## 12.react 生命周期，常用的生命周期

- 挂载：`constructor`、`render`、`componentDidMount`。
- 更新：`getDerivedStateFromProps`、`shouldComponentUpdate`、`render`、`componentDidUpdate`。
- 卸载：`componentWillUnmount`。

## 13.对应的生命周期做什么事

- `componentDidMount` 适合请求数据、订阅事件。
- `shouldComponentUpdate` 可用于性能优化。
- `componentDidUpdate` 处理更新后的副作用。
- `componentWillUnmount` 清理定时器、订阅、事件监听。

## 14.遇到性能问题一般在哪个生命周期里解决

- 类组件常在 `shouldComponentUpdate` 中避免不必要渲染。
- 也可以使用 `PureComponent` 或在函数组件中使用 `React.memo`。

## 15.怎么做性能优化（异步加载组件...）

- 使用 `React.lazy` 和动态 `import()` 做组件懒加载。
- 路由级代码分割，减少首屏 bundle。
- 配合缓存、预加载、虚拟列表和 memo 优化运行时性能。

## 16.写 react 有哪些细节可以优化

- 避免在 render 中创建不必要的新对象和新函数。
- 列表 key 使用稳定 id。
- 状态放在最小必要范围，避免无关组件重渲染。

## 17.React 的事件机制（绑定一个事件到一个组件上）

- React 使用合成事件封装原生事件，屏蔽浏览器差异。
- 大多数事件委托到根容器统一监听，再按组件树派发。

## 18.介绍下事件代理，主要解决什么问题

- 事件代理利用冒泡，把多个子元素事件交给父元素统一处理。
- 主要解决大量事件绑定带来的内存和性能问题，也方便处理动态新增节点。

## 19.前端开发中用到哪些设计模式

- 单例：全局 store、弹窗管理。
- 工厂：根据类型创建组件或策略。
- 观察者/发布订阅：事件中心、状态订阅。
- 代理/装饰器：缓存、权限、埋点包装。

## 20.React/Redux 中哪些功能用到了哪些设计模式

- React 组件组合体现组合模式。
- Context、store 订阅体现观察者模式。
- Redux 中间件体现洋葱模型和责任链思想。
- 高阶组件体现装饰器模式。

## 21.JS 变量类型分为几种，区别是什么

- 基本类型：`undefined`、`null`、`boolean`、`number`、`string`、`symbol`、`bigint`。
- 引用类型：对象、数组、函数等。
- 基本类型复制值，引用类型复制引用地址。

## 22.JS 里垃圾回收机制是什么，常用的是哪种，怎么处理的

- 主流垃圾回收基于标记清除，从根对象出发标记可达对象。
- 不可达对象会被回收。
- 现代引擎还会结合分代回收、增量标记等策略减少停顿。

## 23.一般怎么组织 CSS（Webpack）

- 使用 `css-loader`、`style-loader` 或 `MiniCssExtractPlugin` 处理 CSS。
- 通过 Sass/Less、PostCSS、CSS Modules 或 BEM 组织样式。
- 生产环境抽离 CSS、压缩、加 hash，配合缓存发布。
