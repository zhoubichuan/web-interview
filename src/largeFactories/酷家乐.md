# 十一、酷家乐

## 1.react 生命周期

- 挂载：`constructor`、`render`、`componentDidMount`。
- 更新：`getDerivedStateFromProps`、`shouldComponentUpdate`、`render`、`componentDidUpdate`。
- 卸载：`componentWillUnmount`，用于清理副作用。

## 2.react 性能优化

- 减少无效渲染：`React.memo`、`PureComponent`、`useMemo`、`useCallback`。
- 长列表使用虚拟滚动，列表项使用稳定 `key`。
- 状态下沉或拆分，避免父组件更新牵连过多子组件。

## 3.添加原生事件不移除为什么会内存泄露

- DOM 节点或组件销毁后，事件回调仍可能引用组件实例、闭包变量和 DOM。
- 这些引用会阻止垃圾回收，所以需要在卸载时 `removeEventListener`。

## 4.还有哪些地方会内存泄露

- 未清除的定时器、订阅、WebSocket、全局事件。
- 闭包长期持有大对象。
- 缓存无限增长、 detached DOM 节点仍被引用。

## 5.setInterval 需要注意的点

- 组件卸载或页面离开时要清除。
- 回调执行时间过长可能造成堆积。
- 对精确计时要求高时，应结合时间戳校正误差。

## 6.定时器为什么是不精确的

- JS 是单线程，定时器回调需要等调用栈清空后才能执行。
- 浏览器会对嵌套定时器、后台页面做最小间隔和节流限制。

## 7.setTimeout(1)和 setTimeout(2)之间的区别

- 延迟时间不同，但都不是精确执行时间，只表示最早可进入任务队列的时间。
- 浏览器存在最小延迟限制，实际执行顺序还受主线程任务影响。

## 8.介绍宏任务和微任务

- 宏任务包括 script、setTimeout、setInterval、IO、UI 事件等。
- 微任务包括 Promise.then、MutationObserver、queueMicrotask 等。
- 每个宏任务结束后会清空当前微任务队列，再进入下一个宏任务。

## 9.promise 里面和 then 里面执行有什么区别

- Promise 构造函数中的代码同步执行。
- `then` 回调异步执行，会进入微任务队列。

## 10.介绍 pureComponet

- `PureComponent` 是 React 类组件，内置浅比较 `props` 和 `state`。
- 当浅比较结果未变化时跳过渲染。
- 嵌套对象变化需要保持不可变更新，否则可能漏渲染。

## 11.介绍 Function Component

- 函数组件是用函数定义的 React 组件，接收 props 返回 React 元素。
- 配合 Hooks 可以拥有状态、副作用、缓存等能力。
- 相比类组件更轻量，也更利于逻辑复用。

## 12.React 数据流

- React 采用单向数据流，父组件通过 props 向子组件传递数据。
- 子组件通过回调通知父组件更新状态。
- 复杂场景可用 Context 或状态管理库共享数据。

## 13.props 和 state 的区别

- `props` 由父组件传入，组件自身不应修改。
- `state` 是组件内部状态，可通过 setState 或 Hooks 更新。
- props 更像外部输入，state 更像内部数据。

## 14.介绍 react context

- Context 用于跨层级传递数据，避免 props 层层传递。
- 常用于主题、语言、登录态等全局或半全局数据。
- 高频变化的大状态不宜全部放 Context，否则可能引发大范围更新。

## 15.介绍 class 和 ES5 的类以及区别

- ES5 通常用构造函数和原型实现类。
- ES6 `class` 是原型继承的语法糖，写法更清晰。
- class 默认严格模式，方法不可枚举，必须用 `new` 调用。

## 16.介绍箭头函数和普通函数的区别

- 箭头函数没有自己的 `this`、`arguments`、`super`。
- 箭头函数不能作为构造函数，不能用 `new`。
- 普通函数的 `this` 由调用方式决定，箭头函数继承外层 `this`。

## 17.介绍 defineProperty 方法，什么时候需要用到

- `Object.defineProperty` 可定义对象属性描述符，如 `get`、`set`、`writable`、`enumerable`。
- Vue 2 响应式就是通过它劫持属性读写实现依赖收集和更新通知。

## 18.for..in 和 object.keys 的区别

- `for...in` 会遍历对象自身和原型链上的可枚举属性。
- `Object.keys` 只返回对象自身的可枚举属性数组。

## 19.介绍闭包，使用场景

- 闭包是函数能够访问其定义时外层作用域变量的机制。
- 使用场景包括私有变量、缓存、防抖节流、柯里化和异步回调保留上下文。

## 20.使用闭包特权函数的使用场景

- 通过闭包隐藏私有数据，只暴露受控方法访问或修改。
- 常用于模块封装、计数器、权限校验、缓存读写。

## 21.get 和 post 有什么区别

- GET 通常用于获取数据，参数多放在 URL；POST 常用于提交数据，参数放请求体。
- GET 更容易被缓存和记录，POST 更适合提交敏感或较大数据。
- 语义上 GET 应该是幂等且无副作用，POST 通常会产生服务端状态变化。
