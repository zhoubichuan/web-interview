## 6.\_construct 是什么

## 10.rem、flex 的区别（root em）

- `rem` 是长度单位，相对于根元素 `html` 的字体大小，常用于移动端等比适配。
- `flex` 是布局模型，用于控制主轴、交叉轴上的排列、伸缩和对齐。
- 二者不是同类概念，可以配合使用。

## 11.em 和 px 的区别

- `px` 是 CSS 像素，相对固定。
- `em` 相对于当前元素或继承的字体大小。
- `rem` 相对于根元素字体大小，更适合全局统一缩放。

## 12.React 生命周期

- 挂载：`constructor`、`render`、`componentDidMount`。
- 更新：`getDerivedStateFromProps`、`shouldComponentUpdate`、`render`、`componentDidUpdate`。
- 卸载：`componentWillUnmount`。

## 13.如何去除 url 中的#号

- 使用 history 路由模式，通过 `history.pushState` 管理路径。
- 服务端需要配置 fallback，把所有前端路由都返回入口 HTML。

## 14.Redux 状态管理器和变量挂载到 window 中有什么区别

- Redux 有固定更新流程，状态变化可追踪、可调试。
- window 全局变量可被任意代码修改，缺少约束和通知机制。
- Redux 可通过订阅触发视图更新，全局变量需要额外实现响应逻辑。

## 15.webpack 和 gulp 的优缺点

- webpack 是模块打包器，适合处理模块依赖、代码分割、资源构建。
- gulp 是任务流工具，适合文件复制、压缩、编译等流程编排。
- 现代前端应用通常以 webpack/Vite 等打包器为核心，gulp 更适合辅助自动化任务。

## 16.如何实现异步加载

## 19.并发请求资源数上限（6 个）

- HTTP/1.1 浏览器通常限制同域名并发 TCP 连接数，常见约 6 个。
- HTTP/2 通过多路复用减少这个限制影响。
- 可通过 CDN 多域名、资源合并、懒加载优化请求并发。

## 20.base64 为什么能提升性能，缺点

- 小图片转 base64 可减少一次 HTTP 请求。
- 缺点是体积约增大三分之一，无法单独缓存，也会增加 CSS/JS 文件体积。
- 适合很小且复用少的资源。

## 21.介绍 webp 这个图片文件格式

- WebP 是 Google 推出的图片格式，支持有损、无损、透明和动画。
- 通常比 JPEG/PNG 体积更小。
- 使用时需要考虑浏览器兼容，可通过 `<picture>` 或服务端协商降级。

## 22.介绍 koa2

- Koa2 是 Node.js Web 框架，核心很薄，基于中间件组合。
- 使用 async/await 编写洋葱模型中间件。
- `ctx` 封装 request 和 response，便于处理请求上下文。

## 26.CORS 如何设置

- 服务端设置 `Access-Control-Allow-Origin` 指定允许来源。
- 如需携带 Cookie，还要设置 `Access-Control-Allow-Credentials: true`。
- 非简单请求需处理 OPTIONS，并设置允许的方法和请求头。

## 27.jsonp 为什么不支持 post 方法

- JSONP 依赖 `script` 标签加载跨域脚本。
- `script` 只能发起 GET 请求，无法发送 POST 请求体。

## 30.介绍 Immuable

- Immutable 数据指创建后不直接修改，更新时返回新对象。
- 好处是便于浅比较、状态追踪和撤销回放。
- React/Redux 中常用不可变更新来减少渲染判断成本。

## 31.介绍下 redux 整个流程原理

- 组件 dispatch action。
- middleware 可拦截 action 处理异步或日志。
- reducer 根据旧 state 和 action 返回新 state。
- store 通知订阅者，组件重新获取状态并渲染。

## 32.介绍原型链

- 对象访问属性时，先查自身，找不到就沿 `__proto__` 向上查找。
- 构造函数的 `prototype` 会成为实例的原型。
- 原型链顶端通常是 `Object.prototype`，再往上是 `null`。
